# OkaneTrack 💴

A full-stack personal and company finance tracking web app built with Django 6 and PostgreSQL. Track expenses, income, investments, apply budgeting rules, and get AI-powered financial insights through a chatbot — all inside isolated workspaces.

---

## Features

- **Dual Workspace Modes** — Personal (violet theme) and Company (blue theme) in the same account
- **Expense Tracking** — Log expenses by category, date, and note
- **Income Tracking** — Record income by source
- **Analytics Dashboard** — Spending by category, monthly totals, all-time totals
- **Strategy (50/30/20 Rule)** — Customisable budget rule with burn rate calculation
- **Grow (Investments)** — Track stocks, mutual funds, crypto, and more with P&L calculations
- **AI Advisor (Chatbot)** — Cohere-powered chatbot with real financial context injected per workspace
- **Multi-Workspace** — One user can have unlimited personal and company workspaces
- **50 Automated Tests** — Full test coverage across models, views, auth, and access control

## Database Models

### Workspace
The core container. Every expense, income, and investment belongs to a workspace.

```
user          → ForeignKey(User)
name          → CharField
mode          → CharField (personal | company)
created_at    → DateTimeField
```

### Expense
```
workspace     → ForeignKey(Workspace)
title         → CharField
amount        → DecimalField
category      → CharField
date          → DateField
note          → TextField (optional)
```

### Income
```
workspace     → ForeignKey(Workspace)
title         → CharField
amount        → DecimalField
source        → CharField
date          → DateField
note          → TextField (optional)
```

### BudgetRule
One rule per workspace. Drives the 50/30/20 strategy page.

```
workspace     → OneToOneField(Workspace)
needs_percent → DecimalField (default: 50)
wants_percent → DecimalField (default: 30)
savings_percent → DecimalField (default: 20)
```

### Investment
```
workspace          → ForeignKey(Workspace)
name               → CharField
asset_type         → CharField (stock | mutual_fund | crypto | real_estate | other)
amount_invested    → DecimalField
current_value      → DecimalField
date_invested      → DateField
status             → CharField (active | exited)
note               → TextField (optional)

# Computed properties:
returns            → current_value - amount_invested
returns_percent    → (returns / amount_invested) * 100
is_profitable      → returns > 0
```

## Setup & Installation

### 1. Clone the repository

```bash
git clone https://github.com/yourusername/expense-tracker-django.git
cd expense-tracker-django
```

### 2. Create and activate virtual environment

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Create PostgreSQL database and user

```sql
psql -U postgres

CREATE DATABASE okanetrack;
CREATE USER okanetrack_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE okanetrack TO okanetrack_user;
\q
```

### 5. Create `.env` file

Create a `.env` file in the project root:

```
SECRET_KEY=your-django-secret-key
DEBUG=True
DB_NAME=okanetrack
DB_USER=okanetrack_user
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432
COHERE_API_KEY=your-cohere-api-key
```

### 6. Apply migrations

```bash
python manage.py migrate
```

### 7. Collect static files

```bash
python manage.py collectstatic
```

### 8. Run the development server

```bash
python manage.py runserver
```

Visit `http://127.0.0.1:8000` in your browser.

## What I Learned Building This

- Django's MVT architecture and how views, models, and templates work together
- PostgreSQL integration with psycopg2 and Django ORM queries
- `annotate()` vs `aggregate()` — per-row vs whole-queryset calculations
- Template inheritance with `base.html` and `{% block %}` tags
- `get_object_or_404` for safe workspace ownership enforcement
- `@login_required` decorator and access control patterns
- Writing Django tests with `TestCase`, `Client`, and `unittest.mock` for external APIs
- Reading Django tracebacks — the last line is always the actual error
- Why Python silently allows duplicate function definitions (and how to catch it)
- `STATIC_URL` vs `STATIC_ROOT` — dev serving vs production collectstatic
